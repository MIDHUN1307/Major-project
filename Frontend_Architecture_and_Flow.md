This document provides a technical overview of the **Frontend** of the AI Interview Preparation platform, detailing the file structure, key components, and the user journey through various modules.

## 3. Frontend Internal Workflow

### I. Data Synchronization Flow
The frontend uses a **Dual-Persistence Strategy**:
1.  **Firebase Sync**: On dashboard load, the app fetches `hr_interviews` and `core_results` from the cloud.
2.  **LocalStorage Sync**: At the same time, it reads `aptitude_bandit_state` and `coding_progress` from the browser.
3.  **Aggregation**: `progressMetrics.js` combines these two sources into a single state object to render the dashboard charts.

### II. The Adaptive Test Lifecycle (Aptitude)
1.  **Trigger**: User selects "Adaptive Test."
2.  **Curate**: `generateCuratedTest()` runs in `banditModel.js`. It calculates the "need for practice" score for all topics.
3.  **Render**: `Test.jsx` receives the curated list and starts a 10-minute timer.
4.  **Finalize**: Upon submission, stats are updated in `localStorage` and the UI redirects to `Result.jsx` to show new strong/weak areas.

### III. The Interview Lifecycle (HR Module)
1.  **Access**: User enters `HRInterview.jsx`.
2.  **Capture**: `MediaRecorder API` starts capturing audio blobs.
3.  **Visual Feedback**: `VoiceVisualizer.jsx` uses the `Web Audio API`'s `AnalyserNode` to draw real-time frequency bars.
4.  **Process**: Audio is sent to the FastAPI backend via `Axios`. The frontend awaits the JSON response containing the transcript and score.
5.  **Review**: Results are navigated to `HrSummary.jsx`, where **Recharts** displays the confidence and emotional breakdown.

## 5. Potential Presentation Questions (Frontend Specific)

### Q1: Why use React State over just global variables?
> **Answer**: React State (via `useState` and `useEffect`) ensures the UI stays synchronized with the underlying data. For example, when a question is answered, the progress bar updates instantly without a page reload.

### Q2: How do you handle file uploads in the Core Subject module?
> **Answer**: We use the `FormData` object in JavaScript to wrap the `File` object from the file input. This is then sent to the FastAPI backend using an `Axios` POST request with `multipart/form-data` headers.

### Q3: Explain the real-time audio visualization logic.
> **Answer**: We use the `AudioContext` to create a media stream source from the microphone. We then attach an `AnalyserNode` which provides a frequency byte array. This array is mapped to the height of the bars in the `VoiceVisualizer` component.

### Q4: How is the app's performance optimized?
> **Answer**: We use **Vite** for fast HMR (Hot Module Replacement) and optimized production builds. Additionally, we avoid heavy database calls by caching user progress in `LocalStorage` for the faster Aptitude and Coding modules.
